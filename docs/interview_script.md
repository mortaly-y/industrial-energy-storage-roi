# 项目 3 分钟面试讲解稿

## 1. 项目一句话介绍

我围绕工商业储能解决方案工程师岗位，做了一个 500kW/1MWh 用户侧储能收益测算与方案生成项目，覆盖客户需求分析、容量配置、峰谷套利测算、EMS 策略设计、通讯点表和现场调试 SOP。

## 2. 为什么做这个项目

我过去主要做电气设备现场调试和客户技术支持，具备通讯联调、现场问题排查、客户沟通和验收支持经验。但我没有完整商业储能项目经历，所以我通过这个作品集把岗位关键工作拆开做了一遍，验证自己能否把原有经验迁移到储能解决方案岗位。

## 3. 项目怎么做

我先设定一个制造企业用户场景：客户峰段电费高、最大需量偏高，希望降低综合用电成本。基于这个场景，我配置了 500kW/1MWh 储能系统，接入用户侧低压母线。

然后我用 Python 搭建了一个简化测算模型。输入 24 小时负荷曲线、电价表和储能参数后，模型自动执行谷段充电、峰段放电，并考虑 SOC 上下限、系统效率和防逆流约束，最后输出日收益、年收益、静态回收期和运行曲线。

## 4. 项目输出了什么

这个项目目前输出了几类内容：

1. Python 收益测算代码；
2. 24 小时负荷和电价样例数据；
3. 收益测算 Excel；
4. 负荷曲线、SOC 曲线和充放电曲线；
5. EMS 控制策略说明；
6. EMS-BMS-PCS Modbus 通讯点表；
7. 储能柜现场调试与验收 SOP。

## 5. 我的理解

我理解储能解决方案工程师不是单纯写 PPT，而是要把客户需求转化为可报价、可实施、可调试、可验收的技术方案。

这个岗位需要同时关注四件事：

1. 客户为什么需要储能；
2. 储能配置是否合理；
3. 项目经济性是否成立；
4. 方案到了现场能不能调通和验收。

## 6. 项目边界

这个项目是前期方案测算模型，不是商业级 EMS，也不涉及真实设备控制下发。真实项目还需要结合设备厂家协议、当地电价政策、并网条件、消防规范和项目合同边界进一步细化。

## 7. 和我过往经验的连接

我之前做过电气设备现场调试、通讯配置、动环数据上传、客户培训和问题闭环。储能项目中的 EMS、PCS、BMS、电表通讯联调，本质上也需要类似的现场排查能力，比如设备地址、寄存器偏移、功能码、数据类型、字节序、通讯链路等问题。

所以我的定位不是成熟储能专家，而是有电气现场交付基础、正在向储能解决方案和系统集成方向转型的初中级候选人。
